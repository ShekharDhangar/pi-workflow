# Angle menu — software engineering research

Pick **N distinct angles** up to the depth cap. Prefer variety over duplicate queries.

| Angle | Mission |
|-------|---------|
| **Local — repo (pi-researcher-local)** | What exists in repo; what must not break |
| **Minimal / simple** | Simplest viable pattern |
| **Robust / complex** | Production-grade when requirements are harsh |
| **Industry standard** | How teams actually ship this — docs, blogs, OSS reference impls |
| **Adjacent field** | Borrow from queuing, CAP, stream processing, etc. |
| **Legacy / underpriced** | Brooks, end-to-end, Gall, postmortems |
| **Failure modes** | How this class fails in prod |
| **Cost / ops** | Run cost, on-call, migration |
| **Novel / greenfield** | First-principles from constraints only |
| **Anti-pattern scan** | Known traps in this problem class |
