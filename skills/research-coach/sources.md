# Sources — software engineering research

**Default:** practitioner evidence for shipping decisions — not academic papers unless the brief says so.

| Source type | Examples |
|-------------|----------|
| Official docs | Vendor docs, RFCs, language/framework guides |
| Engineering blogs | Company eng blogs, independent writeups, case studies |
| Community | Reddit, HN, Stack Overflow, Discord archives, forum threads |
| Code & issues | GitHub repos, issues, PRs, reference implementations |
| Postmortems & talks | Incident reports, conference videos, architecture posts |
| Underpriced classics | Brooks, end-to-end argument, Gall — when they apply |

**Papers only when:** user requested them, or the problem is genuinely novel/theoretical and practitioner sources are thin.

## pi-researcher-web task line (copy into every web-angle header)

```
SOURCE PREFERENCE: <from brief — default practitioner web, not papers>
Search: engineering blogs, official docs, GitHub, Reddit/forums, postmortems, reference impls.
Do NOT default to arxiv/academic papers unless SOURCE PREFERENCE says papers.
```
